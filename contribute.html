Add Experiments

Submit hack notes, hardware findings, app ideas, screenshots, and small tutorials that help attendees build with the badge.

1

Build Your Hack

Start with a working badge, note the firmware revision, and keep the first version small.

  • Test the steps from a clean badge state.
  • Record hardware parts and pin usage.
  • Capture photos or screenshots if they clarify the build.
2

Write the Tutorial

Use plain language, exact filenames, expected output, and a rollback step.

  • Include difficulty and estimated time.
  • Show commands or file paths in code blocks.
  • Call out power, battery, and voltage risks.
3

Submit It

Open a pull request with the new page, images, and any source files needed to reproduce it.

  • Use one hack per pull request.
  • Credit external references.
  • Keep screenshots compressed for the web.

Content Checklist

A good entry is short, verified, and useful at a table with limited time.

Do test code, list hardware, document recovery, and include clear photos.

Avoid unverified pinouts, unsafe power instructions, oversized images, and unexplained code dumps.
